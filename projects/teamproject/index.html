Overview

A C++ character controller foundation built for team iteration. The focus is on clean input mapping, responsive movement, and Blueprint-facing gates that allow designers to toggle actions safely.

  • Shows a reliable input pipeline using Enhanced Input actions.
  • Demonstrates camera rig configuration for readable gameplay feedback.
  • Supports future combat and traversal hooks through exposed variables.

Key features

  • ACharacter-based controller with tuned movement and jump arcs.
  • Spring arm and follow camera configuration for consistent framing.
  • Enhanced Input action mapping for move, look, and jump.
  • Blueprint-exposed gating variables for combat and dodge readiness.
  • Component-oriented architecture for quick extension.

System breakdown

Enhanced Input mapping context assigns move, look, and jump actions with clean bindings.
Can_Dodge and Can_Attack toggles exposed to Blueprint gate action availability.

Challenges & solutions

Challenge: Maintaining input clarity

Approach: Action-based mapping with clear naming and Blueprint hooks.

Result: Team members can extend actions without breaking base controls.

Challenge: Camera comfort

Approach: Tuned spring arm length, lag, and rotation constraints.

Result: Movement reads cleanly even during rapid turns.

Challenge: Future-proofing combat hooks

Approach: Blueprint-exposed gating variables and modular component slots.

Result: Designers can iterate without C++ rework.

What I would improve next

  • Add locomotion state blending with animation-driven curves.
  • Implement footstep events for timing and audio cues.
  • Expose camera profiles for different gameplay modes.
