Case study

PaintRunner

Role: Gameplay Programmer Tech: Unity, C#, ShaderLab/HLSL
GitHub Video Build

Overview

A fast prototype exploring runner-style gameplay feel and stylized rendering. The project combines a lightweight gameplay loop with a small shader showcase to test visual feedback and readability.

  • Gameplay loop focused on player feel and obstacle pacing.
  • ShaderLab/HLSL effects used to reinforce movement and impact.
  • UI and architecture kept minimal for easy iteration.

Key features

  • Runner movement tuned for responsiveness and readable pacing.
  • Obstacle spawning system with clear lane structure.
  • Score and combo tracking with minimal UI overhead.
  • ShaderLab/HLSL effects that emphasize speed and paint impacts.
  • Clean scene structure and script organization for quick swaps.

Shader Showcase

A compact set of effects built in ShaderLab/HLSL to support gameplay readability.

System breakdown

Runner loop updates movement, obstacle checks, and score in a predictable tick.
Shader triggers map to impact events for immediate feedback.

Challenges & solutions

Challenge: Maintaining speed readability

Approach: Used shader overlays and tighter camera framing.

Result: Player speed is clear without UI clutter.

Challenge: Obstacle pacing

Approach: Adjusted spawn cadence and lane spacing.

Result: Consistent difficulty ramp with clean flow.

Challenge: Keeping code readable

Approach: Modular scripts with a focused game loop class.

Result: Quick iteration with minimal refactors.

What I would improve next

  • Introduce difficulty curves driven by player performance.
  • Add more shader variants for terrain and obstacle feedback.
  • Build a quick replay capture for feel tuning.
